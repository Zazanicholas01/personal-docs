# Repository setup

Use this page as a starter note whenever you spin up a new codebase.

## Create the repository

1. Pick a clear name that matches the project goal.
2. Add a short description so future-you knows what it is for.
3. Decide whether the repository should start public or private.

## First local steps

```bash
git init
git add .
git commit -m "Initial commit"
```

## Baseline checklist

- Add a README.
- Add a .gitignore.
- Decide on the package manager early.
- Create the first docs page before the project gets busy.
